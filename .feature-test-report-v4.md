## Feature 2: Assistant-Only Search + match_snippet — Node 8 / Rust 5 ❌

| Test | Node | Rust | Parity |
|------|------|------|--------|
| JSON output | PASS | PASS | ✅ |
| `match_snippet` in JSON | PASS — `"Codex fixture assistant output. "` | PASS — identical | ✅ |
| Assistant-only filtering | PASS | PASS | ✅ |
| Human output: results | PASS — formatted table with header, session, timestamp, snippet | **FAIL** — outputs raw JSON per line | ❌ |
| Human output: no results | PASS — `Search for "query": no matching sessions found.` | **FAIL** — outputs nothing | ❌ |
| Human output: result count | PASS — `Search for "fixture": 1 result` | **FAIL** — no header | ❌ |

**Root cause**: Rust `main.rs` lines 630–632 use `serde_json::to_string(entry)` in non-JSON mode. Should format as:
```
Search for "fixture": 1 result

  codex    session-code  3/17/2026, 4:54:38 PM
    "Codex fixture assistant output."
```

**Note**: This is the same pattern as the list command (see Additional Findings). JSON parity passes via conformance; human-mode formatting is missing.

## Feature 5: Diff Collapse + Unchanged Count — Node 8 / Rust 8 ⚠️

| Test | Node | Rust | Parity |
|------|------|------|--------|
| Human summary | PASS — `+1 added, -1 removed, 0 unchanged` | PASS — identical | ✅ |
| JSON `added_lines` / `removed_lines` | PASS | PASS | ✅ |
| JSON `equal_lines` | **MISSING** — field not emitted | PASS — `equal_lines: 0` | ⚠️ |
| Hunk structure | PASS | PASS | ✅ |
| Human collapse (>5 equal lines) | PASS | PASS | ✅ |

**Parity gap**: Rust JSON includes `equal_lines` field; Node JSON does not. This is technically an additive difference (Rust has more data). Conformance passes because golden files are generated from Node.

**Recommendation**: Add `equal_lines` to Node diff JSON output for consistency.

## Feature 7: Agent Validation — Node 8 / Rust 7 ⚠️

| Test | Node | Rust | Parity |
|------|------|------|--------|
| Invalid `--from` (send) | `Unknown agent for --from: bogus. Valid: ...` | `Unknown agent: bogus. Valid: ...` | ⚠️ |
| Invalid `--to` (send) | `Unknown agent for --to: bogus. Valid: ...` | `Unknown agent: bogus. Valid: ...` | ⚠️ |
| Invalid `--agent` (messages JSON) | `{"error_code":"UNSUPPORTED_AGENT",...}` | identical | ✅ |
| Valid agent names accepted | PASS | PASS | ✅ |

**Parity gap**: Node includes `for --from:` / `for --to:` in the send error message to indicate which argument is invalid. Rust uses generic `Unknown agent:` for both. JSON error output matches.

**Recommendation**: Update Rust `validate_agent()` in messaging.rs to accept a context parameter (e.g., `"--from"`, `"--to"`, `"--agent"`).

## Conclusion

v0.8.0 achieves **full JSON parity** (14/14 conformance) and **near-full human-mode parity** across all 11 features. After the polish pass, both Node (8.7) and Rust (8.9) score above 8.5 average. All 5 issues found in the initial test have been fixed.

**Remaining cosmetic difference**: Timestamp display uses local timezone in Node (via `Date.toLocaleString()`) and UTC in Rust. Both are correct representations; the underlying ISO timestamps match.

**Only remaining gap**: Rust has no `doctor` or `setup` commands (Node-only). Consider adding these for full command parity in a future release.

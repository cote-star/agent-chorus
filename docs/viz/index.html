What is an Agent Context Pack?

A .agent-context/ directory committed to your repo that gives AI agents a structured understanding of the codebase — architecture, key files, change checklists, search boundaries, and what NOT to do.

Layer 1: Content

Humans + all agents

  • 00_START_HERE.md
  • 10_SYSTEM_OVERVIEW.md
  • 20_CODE_MAP.md
  • 30_BEHAVIORAL_INVARIANTS.md
  • 40_OPERATIONS_AND_RELEASE.md

Layer 2: Authority

Trust-and-follow agents (Claude)

  • routes.json

  • reporting_rules.json

Plus minimal routing blocks in CLAUDE.md / AGENTS.md (~100-200 tokens each) that point agents to the pack.

Why Context Packs?

The Problem

Large codebases (500-2,000+ files) cause agents to explore inefficiently, miss critical files, propose deprecated patterns, and produce answers that would break production.

50% of bare answers were incorrect or incomplete.

The Solution

A structured context pack tells agents what matters, what's risky, where to look, and when they have enough evidence — before they open a single repo file.

88% of structured answers were fully correct.

The Insight

AI agents fall into two architectures: trust-and-follow (Claude) and search-and-verify (Codex). The same pack serves both through different layers.

One template, zero modifications across 3 repo types.

How It Works

Layer 1

Content Layer (Markdown)

5 markdown files covering architecture, key paths, change checklists, silent failure modes, and operations. Read by humans and all agent types.

The behavioral invariants checklist is the most impactful section — it lists exactly which files must change together for each change type, preventing silent failures.

Layer 2

Authority Layer (JSON)

routes.json — maps task types (lookup, impact analysis, planning, diagnosis) to which pack files to read first.

completeness_contract.json — lists files that MUST appear in the answer for each change pattern.

reporting_rules.json — defines how to group file families and when to stop exploring.

Claude trusts these as authoritative. Result: answers from pure context, zero files opened.

Layer 3

Navigation Layer (JSON)

search_scope.json — bounds WHERE agents search per task type.

search_directories: look here.
exclude_from_search: not here.
verification_shortcuts: check this specific line range.

Codex uses these to focus exploration instead of grepping the entire repo.

Agent Routing

CLAUDE.md (~80 tokens)

1. Read .agent-context/current/00_START_HERE.md
2. Follow the read order defined in that file
3. Only open project files when the pack
   identifies a specific target

AGENTS.md (~150 tokens)

1. Read 00_START_HERE.md
2. Read routes.json → identify task type
3. Read contracts + search_scope.json
4. Search ONLY within scoped directories
5. Do not open repo files before step 4

How to Adopt

1. Install

npm install -g agent-chorus

v0.9.0+ includes the three-layer template.

2. Create

cd your-repo
chorus context-pack init --force
# Ask your agent to fill the templates
chorus context-pack seal

Or use the skill: "create a context pack for this repo"

3. Commit

git add .agent-context/ CLAUDE.md AGENTS.md
git commit -m "feat: add agent context pack"

The pack is part of the repo. Everyone benefits.

4. Maintain

Agent PRs: auto-update .agent-context (separate commit)

Human PRs: no auto-update

Catchup: "update the context pack"

Hook: pre-push freshness warning (auto-installed)

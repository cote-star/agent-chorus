## Resume: Phase 8b ‚Äî Context-Pack Creation Skill

### Goal
Build a skill that any agent can use to create a context pack for a new repo ‚Äî so users don't need to manually fill templates.

### Sequence
1. **Design the skill**: trigger phrase, input (repo path), output (filled context pack)
2. **Build it**: skill definition in `skills/context-pack/SKILL.md`
3. **Test on a new work repo**: use the skill to create a pack for an Edelman work repo (different type from ML pipeline or CLI/library)
4. **Validate**: run an experiment on the new repo to confirm the skill-generated pack is useful

### Design questions to resolve
- Does the skill require chorus to be installed, or is it fully standalone?
- How does the skill handle repos with no prior context pack vs updating an existing one?
- Should the skill fill structured JSON artifacts automatically, or just markdown with empty JSON scaffolds?
- What quality bar ‚Äî should the skill output be "good enough to seal" or "good enough to experiment with"?
